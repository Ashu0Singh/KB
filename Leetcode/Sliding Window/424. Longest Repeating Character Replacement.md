### Optimal Solution

We create a map, and initialise a window. Then we start to expand the window until we get to a point where the `getTheLongest` returns zero, what this usually means is that there is not sequence where we can replace the characters and get a long sequence of the same character then we start to shrink the window and get to a point where we have the max size > 0. 

```java
class Solution {
    public int characterReplacement(String s, int k) {
        char[] arr = s.toCharArray();
        int[] map = new int[26];

        int start = 0, end = 0;
        int max = 0;

        while(start <= end && end < arr.length){
            map[arr[end] - 'A']++;
            int longestSeq = getTheLongest(map, end - start + 1, k);
            max = Math.max(max, longestSeq);

            if(longestSeq == 0){
                map[arr[start] - 'A']--;
                start++;
            }
            end++;
        }

        return max;
    }
    int getTheLongest(int[] map, int windowLength, int k){
        int max = 0;
        for(int i : map){
            int size = i + k;
            if(size  == windowLength) {
                max = Math.max(max, size);
            }
            else if(size > windowLength){
                max =  Math.max(max, windowLength);
            }
        }
        return max;
    }
}
```
