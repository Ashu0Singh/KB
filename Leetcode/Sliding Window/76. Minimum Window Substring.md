## Problem Statement

Given two strings `s` and `t`, find the minimum window in `s` that contains all the characters in `t`, including duplicates. If no such window exists, return an empty string.

**Example:**

- Input: s = "ADOBECODEBANC", t = "ABC"
- Output: "BANC"
- Explanation: The minimum window substring "BANC" contains all characters from "ABC"

## Intuition

The naive approach would be to check all substrings (O(n³)), but we can optimize using the **Two-Pointer Sliding Window** technique:

**Key Insights:**

1. **Expand the window** by moving the right pointer until we have all characters from `t`
2. **Contract the window** by moving the left pointer to find the minimum valid window
3. **Track character frequencies** using two arrays/maps - one for string `t` and one for the current window
4. **Use a counter** to track how many unique characters from `t` have been satisfied with the correct frequency

**Why this works:**

- We avoid checking all substrings by intelligently expanding and contracting
- Once we have a valid window, we try to shrink it from the left to minimize its size
- We keep track of the best (smallest) valid window found
- Time complexity improves from O(n³) brute force to O(n)

## Optimal Solution

```java
class Solution {
    public String minWindow(String s, String t) {
        // Edge case: if t is longer than s, no valid window exists
        if(s.length() < t.length()) return "";

        // Create frequency map for characters in t
        int[] tMap = new int[256];
        int have = 0;  // Count of unique chars from t with required frequency met
        int uniqueCount = 0;  // Total unique chars in t
        for(char ch : t.toCharArray()) {
            if(tMap[ch] == 0) uniqueCount++;
            tMap[ch]++;
        }

        // Create frequency map for current window in s
        int[] sMap = new int[256];
        char[] sChar = s.toCharArray();
        int start = 0;
        int end = 0;

        String ans = "";

        // Expand window with end pointer
        while (start <= end && end < sChar.length){
            // Add character to window
            sMap[sChar[end]]++;

            // If this character now has the required frequency, increment have
            if(sMap[sChar[end]] == tMap[sChar[end]]) have++;

            // Once we have all required characters, try to shrink window
            if(have == uniqueCount){
                while(start <= end && end < sChar.length){
                    if(have < uniqueCount){
                        break;  // Can't shrink anymore
                    }
                    sMap[sChar[start]]--;
                    if(sMap[sChar[start]] < tMap[sChar[start]]) have--;
                    start++;
                }
                // Update answer if this window is smaller
                if(ans.length() == 0 || (end - start + 1) < ans.length())
                    ans = s.substring(start - 1, end + 1);
            }
            end++;
        }
        return ans;
    }
}
```

## Step-by-Step Walkthrough

**Example:** s = "ADOBECODEBANC", t = "ABC"

| Step | Action       | Window                  | sMap Status         | have    | Valid        | Notes              |
| ---- | ------------ | ----------------------- | ------------------- | ------- | ------------ | ------------------ |
| 1-3  | Expand end   | A, AD, ADB              | A:1, D:1, B:1       | 2       | No           | Missing C          |
| 4-6  | Expand end   | ADOB, ADOBE, ADOBEC     | + E:1, C:1          | 3       | **Yes**      | Found valid window |
| 7+   | Shrink start | DOBEC, OBEC, BEC, EC, C | Remove A from start | Still 3 | Check better |
| ...  | Continue     | Eventually find         | BANC                | -       | **Yes**      | Minimum window     |

**Key observations:**

1. First valid window found is "ADOBEC" (length 6)
2. Continue expanding and shrinking to find "BANC" (length 4)
3. Once we pass "BANC", no smaller valid windows exist
4. Return "BANC" as the answer

## Variable Explanations

- **tMap**: Frequency array storing required character counts from string `t`
- **sMap**: Frequency array storing current character counts in our sliding window
- **have**: Counter tracking how many unique characters from `t` have their required frequency satisfied
- **uniqueCount**: Total number of unique characters in `t`
- **start, end**: Two pointers for the sliding window boundaries
- **ans**: Stores the minimum valid window found so far

## Algorithm Flow

```
1. Build frequency map of target string t
2. Use two pointers: start and end
3. EXPAND: Move end pointer right, add characters to window
   - If character frequency matches required, increment have counter
4. OPTIMIZE: When have == uniqueCount (all chars found)
   - Move start pointer right to shrink window
   - Remove characters from left until window becomes invalid
   - Record this window if it's the smallest so far
5. REPEAT: Continue expanding until end reaches the end of s
6. Return the minimum valid window found
```

## Complexity Analysis

- **Time Complexity:** O(n + m) where n = length of `s`, m = length of `t`
  - Each character is visited at most twice (once by end, once by start pointer)
  - Building tMap takes O(m)
- **Space Complexity:** O(1) - Using fixed size arrays of 256 (ASCII characters)

## Key Points to Remember

1. **Two separate maps** - one for target, one for current window
2. **have counter** tracks satisfied unique characters (not total characters)
3. **Expand first** until you have a valid window, then contract
4. **Update answer** whenever you find a valid window smaller than previous
5. **Edge case**: If t is longer than s, immediately return ""
6. **Character frequency matters** - need exact count for each character in t
