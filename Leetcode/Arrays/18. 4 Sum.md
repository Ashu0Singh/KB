### Optimal Solution

So the plan is to sort the array first and then go through the array using a two pointer approach where i and j are the ones a the start of the array, then start is right after j and end it towards the end of the array.

If the sum of the numbers is greater than target, then we can move end a towards start as we know that the array is sorted and it will decrease the sum value. If it's greater we'll have to increase the start to get closer to the target value. And if we find a pair we'll push it into an List. After that we'll have to skip all the duplicates and move to the quadruplet of numbers that is not equal to this.

```java
class Solution {
    public List<List<Integer>> fourSum(int[] nums, int target) {
        List<List<Integer>> answer = new ArrayList<List<Integer>>();
        Arrays.sort(nums);
        for(int i = 0; i < nums.length; i++){
            if(i > 0 && nums[i] == nums[i-1]) continue;
            for(int j = i + 1; j < nums.length; j++){
                if(j > i + 1 && nums[j] == nums[j-1]) continue;
                int start = j + 1, end = nums.length - 1;
                while(start < end){
                    long sum = 0L + nums[i] + nums[j] + nums[start] + nums[end];

                    if (sum < target) start++;
                    else if (sum > target) end--;
                    else {
                        answer.add(Arrays.asList(nums[i], nums[j], nums[start], nums[end]));
                        while(start < end && nums[start] == nums[start + 1]) start++;
                        while(start < end && nums[end] == nums[end - 1]) end--;
                        start++; end--;
                    }
                }
            }
        }
        return answer;
    }
}
```
