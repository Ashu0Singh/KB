[Question](https://leetcode.com/problems/sort-colors/)
### Brute force Solution

Just use Arrays.sort or any other sorting algorithm to do so and the time complexity would be O(n log n). 
### Better Solution

So here we will count all the colours and then put them back in the array in order. Simple!

```java
class Solution {
    public void sortColors(int[] nums) {
        int red = 0;
        int white = 0;
        int blue = 0;
        for(int num : nums){
            if(num == 0){
                red++;
            }
            else if(num == 1){
                white++;
            }
            else{
                blue++;
            }
        }

        for(int i=0;i<red;i++){
            nums[i] = 0;
        }
        for(int i=red;i<white+red;i++){
            nums[i] = 1;
        }
        for(int i= white+red;i<nums.length;i++){
            nums[i] = 2;
        }
        
    }
}
```

### Optimal Solution
Look pretty compact right? It is Dutch National Flag approach.
So low is the base anything before low is 0
Anything from low and before mid is 1
Anything from mid and high is unsorted
Anything after high is 2

You can do the same thing assuming low -> mid is unsorted. Here I’m gonna go mid to high be unsorted. 
So mid is the first position and high is the last. If we find a zero at mid we swap it with the current low position and increase low and mid by one. And we do it until mid and high cross each other that means there are no more elements left in the array.

```java
class Solution {
    public void sortColors(int[] nums) {
        int low = 0;
        int mid = 0;
        int high = nums.length - 1;

        while(mid <= high){
            if(nums[mid] == 0)
                swap(nums, low++ , mid++);
            else if(nums[mid] == 1)
                mid++;
            else
                swap(nums, mid , high--);
        }
    }
    private void swap(int[] nums, int low, int high){
        int temp = nums[low];
        nums[low] = nums[high];
        nums[high] = temp;
    }
}
```
