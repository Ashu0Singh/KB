### Optimal Solution

This solution uses **dynamic programming with space optimization**. Instead of creating a separate DP array, it reuses the input array to store the maximum money that can be robbed up to each house.

**Key Idea:**
At each house `i`, you have two choices:
1. **Rob house i**: Take money from house i + maximum money robbed up to house i-2 (since adjacent houses trigger an alarm)
2. **Skip house i**: Take the maximum money robbed up to house i-1

**Algorithm Walkthrough:**

1. **Initialize**: Start iterating from house 1 (index 1)
2. **For each house i**:
   - `robNow = nums[i]` → Start with robbing current house
   - If `i > 1`, add `nums[i-2]` → Can safely rob 2 houses back
   - If `nums[i-1] > robNow`, update to `robNow = nums[i-1]` → Better to skip current house
   - Store the maximum in `nums[i]`
3. **Return**: `nums[nums.length - 1]` contains the maximum money robbed

**Example**: `nums = [1, 2, 3, 1]`
- House 0: max = 1 (rob house 0)
- House 1: max = 2 (rob house 1, skip house 0)
- House 2: max = 4 (rob house 2 + house 0 = 3 + 1)
- House 3: max = 4 (rob house 3 + house 1 = 1 + 2, but we already have 4)
- Result: 4

**Time Complexity**: O(n) — single pass through array  
**Space Complexity**: O(1) — modifying input in-place

```java
class Solution {
    public int rob(int[] nums) {
        for(int i = 1; i < nums.length; i++){
            int robNow = nums[i];
            if(i > 1) robNow += nums[i - 2];
            if(nums[i - 1] > robNow) robNow = nums[i-1]; 
            nums[i] = robNow;
        }
        return nums[nums.length - 1];
    }
}
```
