### Optimal Solution

This solution extends the linear House Robber problem to handle a **circular arrangement** where the first and last houses are adjacent. The key insight is that robbing both the first and last house is impossible, so we have two scenarios:

**Key Idea:**
Since houses are arranged in a circle, we can't rob both house 0 and house n-1. So we reduce the problem to two linear cases and take the maximum:
1. **Case 1**: Rob houses from index 0 to n-2 (exclude the last house)
2. **Case 2**: Rob houses from index 1 to n-1 (exclude the first house)

**Algorithm:**

1. **Edge case**: If only 1 house, rob it
2. **Try both scenarios**: Compare the results of robbing without the last house vs without the first house
3. **Helper function `robLinear`**: Solves the linear house robber problem using space-optimized DP:
   - `prev2` = max money 2 houses back
   - `prev1` = max money 1 house back
   - `curr` = max(skip current house, rob current + prev2)
   - Slide the window forward through each house
4. **Return**: Maximum of the two scenarios

**Example**: `nums = [5, 3, 4, 11, 2]`
- **Case 1** (exclude last): Rob from [5, 3, 4, 11] → 5 + 11 = 16
- **Case 2** (exclude first): Rob from [3, 4, 11, 2] → 3 + 11 = 14
- Result: 16

**Time Complexity**: O(n) — two linear passes  
**Space Complexity**: O(1) — only using a few variables

```java
class Solution {
    public int rob(int[] nums) {
        int n = nums.length;
        if (n == 1) return nums[0];

        return Math.max(
            robLinear(nums, 0, n - 2),
            robLinear(nums, 1, n - 1)
        );
    }

    private int robLinear(int[] nums, int start, int end) {
        int prev2 = 0;
        int prev1 = 0;

        for (int i = start; i <= end; i++) {
            int curr = Math.max(prev1, prev2 + nums[i]);
            prev2 = prev1;
            prev1 = curr;
        }

        return prev1;
    }
}
```
