[Question](https://leetcode.com/problems/3sum/description/)

### Problem Explanation

Find all **unique triplets** in an array that sum to **zero**. Return a list of lists with distinct combinations (no duplicate triplets).

**Key Challenge:** Avoid returning duplicate triplets while efficiently finding all valid combinations.

### Intuition

This is an extension of Two Sum II. Instead of finding two numbers, we fix one number and find two others using the two-pointer approach:

1. **Sort the array** - enables two-pointer technique and helps skip duplicates
2. **Fix one number** - iterate through each element
3. **Find two others** - use two pointers to find pairs that sum to `-nums[i]`
4. **Skip duplicates** - crucial to avoid returning duplicate triplets

### Why Skip Duplicates?

When we find a valid triplet:

- We increment `start` and decrement `end`
- But if the next elements are the same as the ones we just used, they'll form duplicate triplets
- We skip these duplicates by advancing pointers over identical values

**Example:** `nums = [-1, 0, 0, 1, 1]`

- When we find triplet `[-1, 0, 1]`, the next 0 and next 1 would create the same triplet
- So we skip the duplicate 0's and duplicate 1's

### Example Walkthrough

**Input:** `nums = [-1, 0, 1, 2, -1, -4]`

**Step 1: Sort**

```
[-4, -1, -1, 0, 1, 2]
```

**Step 2: Fix first element and find pairs**

```
i=0, nums[i]=-4:
  Two-pointer search for sum=4 → no valid pairs

i=1, nums[i]=-1:
  start=2, end=5: -1 + 0 + 2 = 1 (too small) → start++
  start=3, end=5: -1 + 1 + 2 = 2 (too small) → start++
  start=4, end=5: -1 + 1 + 2 = 2 (too small) → start++
  (no valid pairs for -1 in this array)

i=2, nums[i]=-1 (skip because nums[i] == nums[i-1])
  Continue to avoid duplicate results

i=3, nums[i]=0:
  start=4, end=5: 0 + 1 + 2 = 3 (too big) → end--
  start=4, end=4: (start < end fails, exit)
  (no valid pairs)

Result: No triplets found (this specific example has none)
```

**Better example:** `nums = [-1, 0, 1, 2, -1, -4]` adjusted to `[-2, 0, 1, 2, -1, 0]`

After sorting: `[-2, -1, 0, 0, 1, 2]`

```
i=0, nums[i]=-2:
  Looking for pairs that sum to 2
  start=1, end=5: -1 + 2 = 1 (too small) → start++
  start=2, end=5: 0 + 2 = 2 ✓
  Found: [-2, 0, 2]
  Increment start, and skip the duplicate 0 at position 3
```

### Code Explanation

```
1. Arrays.sort(nums)
   → Sort to enable two-pointer and duplicate skipping

2. for(int i = 0; i < nums.length - 1; i++)
   → Fix one number at a time

3. if(i > 0 && nums[i] == nums[i-1]) continue;
   → Skip duplicate values for the fixed number
   → Prevents processing same triplet multiple times

4. int start = i + 1, end = nums.length - 1;
   → Two pointers for the remaining array

5. while(start < end)
   → Move pointers based on sum

6. After finding a valid triplet:
   while(start < nums.length && nums[start] == nums[start-1]) start++;
   → Skip all duplicate values of start pointer

   while(end > 0 && nums[end] == nums[end + 1]) end--;
   → Skip all duplicate values of end pointer
```

### Optimal Solution

```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        Arrays.sort(nums);
        for(int i = 0; i < nums.length - 1; i++){
            if( i > 0 && nums[i] == nums[i - 1]) continue;

            int start = i + 1, end = nums.length - 1;

            while(start < end){
                int sum = nums[i] + nums[start] + nums[end];

                if(sum > 0) end--;
                else if (sum < 0) start++;
                else {
                    result.add(Arrays.asList(nums[i],nums[start++],nums[end--]));
                    while(start < nums.length && nums[start] == nums[start-1]) start++;
                    while(end > 0 && nums[end] == nums[end + 1]) end--;
                }
            }
        }
        return new ArrayList(result);
    }
}
```
