### Optimal Solution

**Approach:**
Use binary search to find the minimum element. Since duplicates are allowed, we handle them carefully:

- First, skip duplicate values at the boundaries (start and end).
- Compare the mid element with the end element:
  - If `nums[mid] > nums[end]`, the minimum is in the right half, so move start forward.
  - If `nums[start] < nums[mid]`, the minimum is in the left half (or at the boundary), so move end backward.
  - Otherwise (when duplicates prevent clear determination), just move end backward to reduce the search space.
- Track the minimum throughout the process and return it at the end.

**Time Complexity:** O(n) worst case (when all elements are duplicates), O(log n) average case.

```java
class Solution {
    public int findMin(int[] nums) {
        int start = 0, end = nums.length - 1, min = nums[0];

        while(start < end && nums[start] == nums[end])
            start++;

        while(start <= end){
            int mid = (start + end) / 2;
            min = Math.min(min, nums[mid]);

            if(nums[mid] > nums[end]) {
                while(mid < end && nums[mid] == nums[mid + 1])
                    mid++;
                start = mid + 1;
            }
            else if(nums[start] < nums[mid]){
                while(mid > start && nums[mid - 1] == nums[mid])
                    mid--;
                end = mid - 1;
            } else {
                end--;
            }
        }

        return min;
    }
}
```
