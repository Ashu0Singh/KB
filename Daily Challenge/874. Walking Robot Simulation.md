### Optimal Solution

So It's pretty simple for this one, we can move in 4 directions North, East, South and West. I've defined the direction array accordingly. If we're moving in North only `x` will change and other respectively. So the simulation works perfectly, we just need to make sure we're handling the obstacles properly. Since it's know to us that the coordinates can go anything between -30000 to 30000 so our hash function should be able to suffice that. Thats why, 600001 X y + x. What we're doing is that we're moving one by one, since every command can only move the robot from 1 to 9, so we are doing it in constant time. And hence our time complexity is O(n + m)

```java
class Solution {
    public int robotSim(int[] commands, int[][] obstacles) {
        int d = 0;
        int[][] direction = {{0,1}, {1,0}, {0,-1}, {-1,0}};

        HashSet<Integer> set = new HashSet<>();

        int x = 0, y = 0;
        int max = 0;
        for(int[] obstacle : obstacles) set.add(obstacle[0] * 600001 + obstacle[1]);

        for(int command : commands){
            if(command == -1){
                d = (d + 1) % 4;
            } else if (command == -2){
                d = (d - 1) < 0 ? 3 : d - 1;
            } else {
                int dY = direction[d][0];
                int dX = direction[d][1];

                for(int i = 1; i <= command; i++){
                    if(set.contains((y + dY) * 600001 + (x + dX))) break;

                    x = x + dX;
                    y = y + dY;
                }

                max = Math.max(max , ((y * y) + (x * x)));
            }
        }

        return max;
    }
}
```
